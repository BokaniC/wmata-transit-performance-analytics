# 📖 Executive Summary

The Washington Metropolitan Area Transit Authority (WMATA) operates a complex rail and bus network across multiple jurisdictions, generating large volumes of operational, ridership, and safety data.

This project developed a relational database and analytical reporting framework to consolidate data from multiple operational systems and support data-driven decision-making related to ridership growth, service reliability, safety performance, and resource allocation.

Using SQL and Tableau, our team transformed siloed transit data into an integrated analytical environment capable of supporting operational planning and performance monitoring.

---

# 🎯 Business Problem

WMATA relies on data from multiple systems including:

- Faregate activity logs
- Bus performance records
- Safety incident reports
- Route and station reference data
- Calendar and operational reporting systems

Because these datasets exist in separate systems, obtaining a unified view of transit performance can be challenging.

The objective of this project was to create a centralized analytical framework capable of answering critical business questions related to:

- Ridership trends
- Service reliability
- Safety performance
- Resource allocation

# 🏗️ Database Architecture

The project was built around five core entities:

- Station
- BusRoute
- CalendarDate
- RailPerformance
- BusPerformance

The database integrated ridership, reliability, and safety metrics into a unified structure that supports operational reporting and analytical queries.

---

# 🔍 Analysis Performed

The project addressed four major business objectives:

## 1️⃣ Ridership Analysis

Analyzed ridership patterns from 2022–2024 across rail stations and bus routes.

Key questions included:

- Which stations have the highest ridership?
- Which bus routes generate the most boardings?
- Where should operational resources be concentrated?

---

## 2️⃣ Service Reliability Analysis

Evaluated on-time performance across bus routes to assess service reliability and operational consistency.

---

## 3️⃣ Safety & Incident Analysis

Examined crime incidents and customer injuries across stations, jurisdictions, and rail lines.

The analysis identified locations where safety risks exceeded expectations based on ridership levels.

---

## 4️⃣ Resource Allocation Analysis

Developed ranking frameworks to help prioritize:

- Staffing
- Maintenance
- Infrastructure investments
- Safety interventions

# 💼 Business Recommendations

Based on the analysis, WMATA could:

✅ Increase staffing and support at high-demand stations

✅ Prioritize maintenance at critical transit hubs

✅ Deploy targeted safety interventions at incident hotspots

✅ Use ridership patterns to improve route planning and fleet allocation

✅ Expand KPI monitoring through automated reporting dashboards
